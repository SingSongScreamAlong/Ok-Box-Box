- No position, gaps, or timing data
- No graphs, charts, or scrolling data

- Team pit wall (live, for strategists)
- Post-session analytics (delayed, for driver review)
- League/broadcast surfaces (external)

- **Type:** Simple colored dots with labels

- **Content:** Last AI engineer message
- **Duration:** Visible for 10 seconds, then fades

- **Type:** Large button, hold to activate
- **States:** Idle, Recording, Processing, Speaking
- **Keyboard:** Configurable hotkey (default: V)

- No continuous data polling to driver surface
- WebSocket connection for voice and status only

### Components

#### Race State Pane
- Current position and total cars
- Current lap and total laps
- Gaps to leader, car ahead, car behind
- Session status (green/yellow/red flag)

#### Strategy Timeline
- Visual timeline showing current lap position
- Pit window marker (fuel-based)
- Fuel critical marker
- AI recommendation text

#### Car Status Pane
- Fuel level with progress bar
- Estimated laps remaining
- Tire compound and wear
- Damage indicators (if any)

#### Opponent Intel Table
- Sortable table of nearby competitors
- Gap (positive = ahead, negative = behind)
- Trend arrows (gaining/losing/stable)
- Current pace (rolling average)

#### Event Log
- Chronological list of significant events
- Sector times, flags, pit stops, position changes
- Filterable by event type

# 7. DESIGN SYSTEM

## 7.1 Design Philosophy

**"Racing DDU Inspired Dark Mode"**

The design draws inspiration from professional racing Digital Dash Units (DDUs) and pit wall displays:
- **High contrast** for readability in any lighting
- **Color-coded information** for instant recognition
- **Minimal chrome** â€” data takes center stage
- **Consistent dark theme** â€” never light mode

## 9.2 Layout Adaptations

### Driver HUD
- **Desktop:** Full layout as designed
- **Tablet:** Stacked panels, reduced spacing
- **Mobile:** Not supported (requires desktop for racing)

### Pit Wall Dashboard
- **Desktop:** 2x2 grid layout
- **Tablet:** 2x2 grid, smaller fonts
- **Mobile:** Single column stack

### Race Control
- **Desktop:** Side-by-side timing and incidents
- **Tablet:** Tabbed interface
- **Mobile:** Single panel with navigation

### Broadcast Director
- **Desktop:** Full control panel
- **Tablet:** Simplified controls
- **Mobile:** Not recommended

## 9.3 Touch Considerations

- Minimum touch target: 44x44px
- Increased spacing on touch devices
- Swipe gestures for panel navigation
- Long-press for context menus
